## Overview

You use Vivek Dinanath Documentation to create structured, collaborative spaces for your projects. This guide covers adding content, organizing projects with folders, and sharing with teams. Start by navigating to your workspace dashboard to begin editing.

<Callout kind="tip">
  Enable version history in project settings to track changes automatically.
</Callout>

## Adding and Structuring Content

Create new pages and structure them with Markdown and MDX components for rich documentation.

<Steps>
  <Step title="Create a New Page" icon="plus">
    Click the `+ New Page` button in the sidebar. Enter a title like "API Reference" and select MDX format.
  </Step>
  <Step title="Add Headings and Text">
    Use `## Heading` for sections. Write concise paragraphs in active voice.
  </Step>
  <Step title="Insert Components">
    Add JSX components like `<Callout>` for notes. Preview changes live.
  </Step>
  <Step title="Save and Publish">
    Hit `Publish` to make it live. Use `Save Draft` for unfinished work.
  </Step>
</Steps>

## Project Organization and Folders

Organize content hierarchically with folders to keep large projects manageable.

<Columns cols={3}>
  <Card title="Root Folders" icon="folder" href="#">
    Store top-level docs like "User Guide" and "API".
  </Card>
  <Card title="Subfolders" icon="folders" href="#">
    Nest sections: `guides/quickstart/`.
  </Card>
  <Card title="Search and Tags" icon="search" href="#">
    Tag pages with `["feature", "guide"]` for quick filtering.
  </Card>
</Columns>

| Folder Type | Use Case | Permissions |
|-------------|----------|-------------|
| Public     | Open docs | Read for all |
| Private    | Internal  | Owner only  |
| Shared     | Team      | Invite-only |

## Collaboration and Sharing Options

Invite teammates and control access levels.
